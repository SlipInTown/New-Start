Расписать в текстовом документе плюсы и минусы изученных структур данных и расписать примеры в которых нужно будет использовать тот или иной контейнер.

Класс List<T> 

Позволяет добавлять, удалять и изменять элементы в четкой последовательности.
Плюсы: Отлично подходит для вставки и удаления элемента. Минусы: Плох для индексации элементов и их поиска. Средняя сложность по памяти


Класс HashSet<T> и SortedSet<T>

содержит неповторяющуюся коллекцию элементов так, как работает по уникальному хеш-коду элементов и следственно в больших объемах данных имеет куда больше прирост скорости.
Плюсы: Хорошо подходит для вставки, поиска и удаления элемента. Минусы: В худшем случае имеет ужасную временную сложность для всех операций. Средняя сложность по памяти

Класс ObservableCollection<T>

можно подписаться с помощью данной коллекции и сама коллекция будет уведомлять всех слушателей об изменении.


LinkedList<T> 

Связный список представляет собой длинную цепочку в которой каждый элемент хранит ссылку на два смежных элемента (последующий и предыдущий).
Плюсы: Отлично подходит для поиска максимума и удаления элемента. Минусы: Плох для вставки элемента и слияния.

Структура KeyValuePair<TKey, TValue> 

Используется когда нужно искать значения по ключу (первый элемент структуры), через него можно быстро находить, изменять значения.
Плюсы: Отлично подходит для поиска и изменения элемента. Минусы: Плох для вставки элемента и слияния.

Класс Dictionary<TKey, TValue> 

То, же самое что и выше.
Плюсы: Отлично подходит для вставки, добавления и удаления элемента. Минусы: Плох для всех других случаев при коллизии.


Класс Queue<T>

Очередь элементов организованная по методу First In First Out (FIFO), первый элемент всегда будет же первый удалятся из очереди.
Плюсы: Отлично подходит для вставки, добавления и удаления элемента. Минусы: Нельзя взять не первый элемент необходимо расчистить очередь, чтобы дойти до нужного

Класс Stack<T>

Как очередь, но наоборот работающая по методу Last In First Out (LIFO), последний элемент будет удалятся из очереди первым.
Плюсы: Отлично подходит для вставки, добавления и удаления элемента. Минусы: Нельзя взять не последний элемент необходимо расчистить очередь, чтобы дойти до нужного
